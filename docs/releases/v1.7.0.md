# arxgen v1.7.0

## Highlights

- Added schema upgrade warnings for removed fields, type changes, nullability changes, and default value changes.
- Added `--force` handling for risky schema upgrade apply runs.
- Improved SQL parsing for indexes, unique indexes, enum fields, decimal and varchar metadata, composite primary keys, and many-to-many join tables.
- Added focused test coverage for v1.7 SQL metadata and risky schema upgrade previews.

## Migration Notes

Run `arxgen upgrade schema --dry-run` before applying changed SQL schemas. Risky upgrades now require `--force`; arxgen still does not delete fields or rewrite existing field types automatically.

## Breaking Changes

Risky schema upgrade apply runs now fail unless `--force` is passed.
